## 3. 清洗操作明细

### 3.1 缺失值处理

| 列名 | 处理方法 | 依据 | 处理行数 |
|------|---------|------|---------|
| `payment_method` | 众数填充（"credit_card"） | 分类变量，众数代表最常见支付方式 | 108 |
| `customer_rating` | 中位数填充（4.2） | 评分呈左偏分布，中位数更稳健 | 195 |
| `amount` | 按 `price × quantity` 推算补齐 | 存在计算关系，可精确还原 | 55 |
| `quantity` | 中位数填充（2） | 离散整数变量，中位数稳健 | 32 |
| `category` | 根据 `product_id` 关联商品主表补齐 | 业务关联推断 | 24 |
| `product_id` | 标记为 `-1`（未知商品）并单独审查 | 无法推断，保留但不丢弃 | 13 |
| `shipping_days` | 中位数填充（3） | 整数变量，中位数代表典型配送时间 | 45 |

### 3.2 重复行处理

| 操作 | 数量 | 处理方式 |
|------|------|---------|
| 完全重复行删除 | 7 | 保留第一条，删除后续完全一致行 |
| `transaction_id` 重复去重 | 3 | 保留最早记录，删除重复拉取的副本 |
| 组合键重复（同用户同时） | 21 | 标记为疑似重复，保留第一条，其余移入待审查表 |

### 3.3 异常值修正

| 异常类型 | 处理方式 | 处理行数 |
|---------|---------|---------|
| `amount` ≤ 0 | 标记为缺失，按 `price × quantity` 重新计算 | 12 |
| `amount` > 50,000 | 缩尾处理（Winsorize），截断至 99.5% 分位数（¥28,930） | 8 |
| `quantity` ≤ 0 | 标记为缺失，由`amount / price` 向上取整推算 | 5 |
| `quantity` > 100 | 缩尾处理至 99% 分位数（12件） | 3 |
| `price` ≤ 0 | 由 `amount / quantity` 推算（平均单价） | 4 |
| `customer_rating` < 1.0 | 缩尾至 1.0（可能是误输入） | 6 |
| `customer_rating` > 5.0 | 缩尾至 5.0 | 4 |
| 未来日期 | 修正为采集日期（2026-05-18） | 2 |
| 过早日期 | 删除（3行，占 0.03%，影响极小） | 3 |
| `shipping_days` > 30 | 缩尾处理至 99% 分位数（14天） | 7 |
