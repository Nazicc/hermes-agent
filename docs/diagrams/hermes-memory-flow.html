Hermes Agent — Memory Flow Architecture

Layered memory system: L0 injected context → L3 evolution signals, unified by memory_recall()

Write Paths

  • • memory tool → L0 (MEMORY.md) + L2 dual-write
  • • viking_remember → L2 (OpenViking KB)
  • • mcp_hindsight_retain → Hindsight graph
  • • session_search → L1 queries across sessions
  • • evolve_server → L3 auto-skill creation

Read Paths

  • • L0: auto-injected every turn (no tool call)
  • • memory_recall → unified L0-L3 query
  • • viking_search/read → L2 semantic retrieval
  • • mcp_hindsight_recall → graph multi-strategy
  • • session_search → FTS5 + LLM summarization

Key Architecture Decisions

  • • L0→L2 dual-path: memory writes to both instantly
  • • Progressive Disclosure: AGENTS.md → pointer
  • • L2 permanent, L1 ephemeral per-session
  • • Hindsight for graph patterns, not raw storage
  • • memory_recall() deduplicates across layers

Memory Lifecycle

  • • Session start: L0 injected · L1 fresh · L2 warm
  • • During session: memory tool writes · viking_remember
  • • Context compression: L1 summarized
  • • Session end: session_search persisted to L1 logs
  • • Cron: daily evolution cycle (02:00) pushes L3
